<requirements>
  1. The website must have a simple and intuitive interface for manual entry of game scores.
  2. The website must generate a unique view-only link for each game, which can be shared with others.
  3. The view-only link page must display the current score board in real-time.
  4. The website must store game history, including the date, time, and scores of each game.
  5. The website must have a "clear history" button to remove all stored game history.
  6. The website must handle errors and display error messages to the user.
  7. The website must be fully responsive and work on mobile devices.
  8. The website must have a loading animation to indicate when data is being loaded or updated.
</requirements>
<constraints>
  1. The website must not allow users to edit or delete game history.
  2. The website must not allow users to access or modify the database directly.
  3. The website must not use any third-party libraries or APIs for authentication or authorization.
  4. The website must not use any client-side storage for sensitive data.
  5. The website must comply with all relevant web development best practices and security guidelines.
</constraints>
